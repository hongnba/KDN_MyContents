# 새 기사 Ollama 분석 - 빠른 가이드

## 📝 한 줄 요약
**URL과 원문만 있으면 Ollama가 자동으로 요약, 키워드, 감성분석을 해줍니다!**

---

## 🎯 필요한 정보 (최소)

새로운 기사를 분석하려면 **7가지 필수 정보**가 필요합니다:

| # | 필드 | 설명 | 예시 |
|---|------|------|------|
| 1 | `url` | 기사 URL | `https://www.yna.co.kr/view/AKR20251021023400003` |
| 2 | `title` | 기사 제목 | `한전 송전망 건설사업 절반이 '지연'...` |
| 3 | `contentsRaw.contents` | 기사 본문 (원문) | `박정 의원 자료…송전선로 48%·변전소...` |
| 4 | `contentsOrgId` | 기관 코드 | `A0013` (연합뉴스) |
| 5 | `contentsOrgName` | 기관명 | `연합뉴스` |
| 6 | `categoryId` | 카테고리 코드 | `B0001` (뉴스) |
| 7 | `categoryName` | 카테고리명 | `뉴스` |

**선택 정보**: `pubDt` (발행일시)

## 🚀 3단계로 실행하기

### 1️⃣ MongoDB에 기사 추가

```bash
cat > /tmp/article.json << 'EOF'
{
  "title": "기사 제목을 여기에",
  "url": "https://www.example.com/news/123",
  "contentsOrgId": "A0013",
  "contentsOrgName": "연합뉴스",
  "categoryId": "B0001",
  "categoryName": "뉴스",
  "rawCollectSucYN": "Y",
  "contentsRaw": {
    "contents": "기사 본문 전체를 여기에..."
  },
  "metaSucYN": "N",
  "lookCount": 0,
  "likeIds": [],
  "disLikeIds": []
}
EOF

docker exec -i ksubscribe_mongodb mongo mycontents --eval \
  "db.contents.insertOne($(cat /tmp/article.json))"
```

### 2️⃣ Ollama 분석 실행

```bash
docker exec python_new timeout 300 python3 /app/run_single_url_ollama.py \
  "https://www.example.com/news/123"
```

### 3️⃣ 결과 확인

```bash
docker exec -i ksubscribe_mongodb mongo mycontents --eval \
  'db.contents.findOne({"url": "https://www.example.com/news/123"}).contentsMeta'
```

## 📊 Ollama가 자동으로 생성하는 것

Ollama는 다음 5가지를 자동으로 분석합니다:

| 항목 | 설명 | 결과 예시 |
|------|------|----------|
| 1️⃣ **검증** | 키워드 관련성 확인 | 에너지 26%, 전력 17% |
| 2️⃣ **짧은 요약** | 1줄 요약 | "한국전력의 송·변전설비 건설사업이 절반 이상 지연..." |
| 3️⃣ **긴 요약** | 5줄 이상 상세 요약 | "박정 의원은 한전으로부터..." |
| 4️⃣ **키워드** | 핵심 키워드 추출 | ["송전망", "재생에너지", "전력 수급"] |
| 5️⃣ **감성분석** | 긍정/부정/중립 비율 | 긍정 0%, 부정 90%, 중립 10% |

**소요 시간**: 약 60-80초

## 🔑 자주 사용하는 코드

### 기관 코드 (contentsOrgId)
```
A0001 = 산업통상자원부
A0013 = 연합뉴스
```

### 카테고리 코드 (categoryId)
```
B0001 = 뉴스
B0012 = 공고
```

더 많은 코드는 MongoDB 조회:
```bash
# 기관 코드 목록
docker exec -i ksubscribe_mongodb mongo mycontents --eval \
  'db.contents_org.find({}, {orgId:1, orgName:1})'

# 카테고리 코드 목록  
docker exec -i ksubscribe_mongodb mongo mycontents --eval \
  'db.contents_org_category.find({}, {cateId:1, cateName:1})'
```

## ⚡ 실제 사용 예시

### 연합뉴스 기사 추가 및 분석

```bash
# 1. 기사 정보 준비
URL="https://www.yna.co.kr/view/AKR20251113123400003"
TITLE="새로운 기사 제목"
CONTENT="기사 본문 전체..."

# 2. MongoDB 삽입
cat > /tmp/yna_article.json << EOF
{
  "title": "${TITLE}",
  "url": "${URL}",
  "contentsOrgId": "A0013",
  "contentsOrgName": "연합뉴스",
  "categoryId": "B0001",
  "categoryName": "뉴스",
  "rawCollectSucYN": "Y",
  "contentsRaw": {"contents": "${CONTENT}"},
  "metaSucYN": "N",
  "lookCount": 0,
  "likeIds": [],
  "disLikeIds": []
}
EOF

docker exec -i ksubscribe_mongodb mongo mycontents --eval \
  "db.contents.insertOne($(cat /tmp/yna_article.json))"

# 3. Ollama 분석
docker exec python_new timeout 300 python3 /app/run_single_url_ollama.py "${URL}"
```

## ❓ FAQ

**Q: URL만 있으면 자동으로 본문을 크롤링해주나요?**
A: 아니요. 현재 스크립트는 **이미 MongoDB에 저장된 기사**만 분석합니다. 본문은 직접 준비해야 합니다.

**Q: 기관 코드를 모르면 어떻게 하나요?**
A: `A0013` (연합뉴스)를 기본값으로 사용하거나, MongoDB에서 조회하세요.

**Q: 같은 기사를 다시 분석할 수 있나요?**
A: 네. `metaSucYN`을 `"N"`으로 변경 후 재실행하면 됩니다.

**Q: 오래 걸리는 이유는?**
A: Ollama가 LLM 모델을 사용하여 5가지 분석을 순차적으로 실행하기 때문입니다 (총 60-80초).
