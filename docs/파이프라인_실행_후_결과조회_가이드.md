# 파이프라인 실행 후 결과 조회 가이드

## 📋 개요

`main_collect_and_scrapping.py` 실행 후, MongoDB `contents` 컬렉션에서 해당 실행으로 생성된 문서만 조회하는 방법을 설명합니다.

## ✅ 1번 질문 답변: 실행 시 발생한 문제

### 확인된 문제
**두 번 실행되는 문제만 있었습니다.**

- ✅ **정상 동작**: 모든 URL이 성공적으로 스크래핑되고 분석됨

#### STEP 6: 시간대별 분포 확인

```bash
docker exec ksubscribe_mongodb mongo --quiet mycontents --eval "
var startTime = ISODate('2025-11-13T05:43:05.326Z');

print('=== 시간대별 문서 생성 분포 ===');
print('');

var pipeline = [
  {\$match: {metaAnalyzeDt: {\$gte: startTime}}},
  {\$project: {
    hour: {\$hour: '\$metaAnalyzeDt'},
    minute: {\$subtract: [{\$minute: '\$metaAnalyzeDt'}, {\$mod: [{\$minute: '\$metaAnalyzeDt'}, 10]}]}
  }},
  {\$group: {
    _id: {hour: '\$hour', minute: '\$minute'},
    count: {\$sum: 1}
  }},
  {\$sort: {'_id.hour': 1, '_id.minute': 1}}
];

db.contents.aggregate(pipeline).forEach(function(doc) {
  var timeSlot = doc._id.hour + ':' + (doc._id.minute < 10 ? '0' : '') + doc._id.minute;
  var endMinute = doc._id.minute + 10;
  print(timeSlot + ' ~ ' + doc._id.hour + ':' + (endMinute < 10 ? '0' : '') + endMinute + ' : ' + doc.count + '개');
});
"
```

**실제 실행 결과:**
```
=== 시간대별 문서 생성 분포 ===

5:50 ~ 5:60 : 7개
6:00 ~ 6:10 : 10개
6:10 ~ 6:20 : 10개
6:20 ~ 6:30 : 1개
```

**분석**: 
- 첫 번째 실행: 5:50~6:10 (14개)
- 두 번째 실행: 6:00~6:20 (14개)
- 중복 구간(6:00~6:10): 두 실행이 겹침
- **총 28개 = 중복 실행으로 인한 결과**

## 🔧 자동화 스크립트

위 과정을 자동화한 스크립트를 제공합니다.

### 사용법

```bash
# 1. 실행 전 - 시작 시간을 파일에 기록
echo "START_TIME=$(date -u +"%Y-%m-%dT%H:%M:%S.%3NZ")" > /tmp/pipeline_run.env

# 2. 파이프라인 실행
docker exec ksubscribe_python_unified python3 /app/docker_shell/main_collect_and_scrapping.py

# 3. 실행 후 - 결과 조회
source /tmp/pipeline_run.env
docker exec ksubscribe_mongodb mongo --quiet mycontents --eval "
var startTime = ISODate('$START_TIME');
var count = db.contents.find({metaAnalyzeDt: {\$gte: startTime}}).count();
print('실행 시작: ' + startTime);
print('생성된 문서 수: ' + count);
print('');
print('=== URL 목록 ===');
var idx = 0;
db.contents.find(
  {metaAnalyzeDt: {\$gte: startTime}},
  {url: 1, metaAnalyzeDt: 1, _id: 0}
).sort({metaAnalyzeDt: 1}).forEach(function(doc) {
  idx++;
  print(idx + '. ' + doc.url);
});
"
```

## 📌 핵심 필드 설명

| 필드 | 설명 | 예시 |
|------|------|------|
| `metaAnalyzeDt` | LLM 분석이 완료된 시간 (UTC) | `2025-11-13T05:53:37.933Z` |
| `rawCollectDt` | 웹에서 원본 수집한 시간 (UTC) | `2025-11-13T05:53:30.123Z` |
| `url` | 수집·분석한 URL | `https://example.com/news/123` |
| `metaSucYN` | 메타 분석 성공 여부 | `Y` / `N` |
| `contentsMeta.shortSummary` | LLM이 생성한 짧은 요약 | `한국전력 주가는 ...` |
| `contentsMeta.keywords` | 추출된 키워드 배열 | `["한국전력", "주가"]` |

## ⚠️ 주의사항

1. **UTC 시간 사용**: MongoDB는 모든 날짜를 UTC로 저장합니다. 한국 시간(KST)은 UTC+9입니다.
   ```bash
   # UTC → KST 변환 예시
   # UTC: 2025-11-13T05:43:05Z
   # KST: 2025-11-13 14:43:05 (9시간 더함)
   ```

2. **타임스탬프 정확도**: 파이프라인 시작 **직전**의 시간을 기록해야 새로 생성된 문서만 조회됩니다.

3. **중복 실행 문제**: 현재 코드는 같은 큐를 두 번 처리합니다. 중복 방지를 원하면 별도 패치가 필요합니다.
   - 해결 방법: `/docs/second_run_explanation_20251113.md` 참조

4. **큐 상태 확인**: 실행 전후 큐 상태를 확인하려면:
   ```bash
   docker exec ksubscribe_mongodb mongo --quiet mycontents --eval "db.contents_queue.find({}).count()"
   ```
