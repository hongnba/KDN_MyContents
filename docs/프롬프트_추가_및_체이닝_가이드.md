### 3.2 상세 작업 내용

#### STEP 1: `analysis_ollama_base.py` - 프롬프트 템플릿 정의

```python
# 위치: ksubscribe_server/analysis/analysis_ollama_base.py
# 기존 프롬프트들 아래에 추가

class AnalysisOllamaBase:
    # ... 기존 프롬프트들 ...
    
    # ✅ 새 프롬프트 추가 (예: 키워드 상세 분석)
    question_keyword_detail = f"""
    기사 : [contents]
    기관 : [organization]
    
    위 기사에서 추출된 키워드: [extracted_keywords]
    
    각 키워드에 대해 상세 분석을 수행해 줘.
    
    출력은 아래 JSON 형식으로만 해.
    {{
        "keyword_detail_analysis": [
            {{
                "keyword": "키워드1",
                "importance": (1~10, int),
                "context": "해당 키워드가 언급된 맥락"
            }}
        ]
    }}
    """
```

**⚠️ 주의사항:**
- JSON 중괄호는 `{{`, `}}` 로 이스케이프 필요 (f-string 내부)
- 플레이스홀더는 `[변수명]` 형식으로 정의
- 변수명은 다른 프롬프트와 충돌하지 않도록 고유하게 작명

#### STEP 2: `analysis_ollama_generate.py` - 호출 로직 추가

```python
# 위치: ksubscribe_server/analysis/analysis_ollama_generate.py
# analysis_main() 함수 내부에 추가

def analysis_main(self, content, pred_keyword_list, org_name_list, 
                  mycontents_logger:logging.Logger, queueContent:ContentsQueueVO):
    # ... 기존 로직 ...
    
    # ✅ 새 프롬프트 호출 추가
    # 1) 플레이스홀더 치환
    new_question_keyword_detail = self.question_keyword_detail \
        .replace("[organization]", str(orgName) if orgName else "") \
        .replace("[contents]", content) \
        .replace("[extracted_keywords]", str(extracted_keywords))  # A 프롬프트 결과 사용 가능
    
    # 2) LLM 호출
    result_keyword_detail = self.chat_ollama._client.generate(
        model=CONF.OLLAMA_MODEL,
        prompt=new_question_keyword_detail,
        format=(None if "gpt-oss" in CONF.OLLAMA_MODEL else "json")
    )
    
    # 3) JSON 파싱
    is_success, result_keyword_detail_json = self.json_load(result_keyword_detail, mycontents_logger)
    
    # 4) VO 객체에 매핑 (assemble 함수 호출)
    self.assemble_keyword_detail_to_model(queueContent, orgName, result_keyword_detail, 
                                          result_keyword_detail_json, contentsMetaResult, mycontents_logger)
```

**새 assemble 함수 추가:**

```python
def assemble_keyword_detail_to_model(self, queueContent, orgName, result, result_json, 
                                      contentsMetaResult, mycontents_logger) -> bool:
    """새 프롬프트 결과를 모델에 조립"""
    try:
        # 기존 SentimentInfo 찾기 또는 새로 생성
        target_sentiment = None
        for sentiment in contentsMetaResult.contentsMeta.sentiments:
            if sentiment.orgName == orgName:
                target_sentiment = sentiment
                break
        
        if target_sentiment is None:
            target_sentiment = SentimentInfo()
            target_sentiment.orgName = orgName
            contentsMetaResult.contentsMeta.sentiments.append(target_sentiment)
        
        # ✅ 새 필드 매핑
        target_sentiment.keyword_detail_analysis = result_json.get("keyword_detail_analysis", [])
        
        return True
    except Exception as e:
        mycontents_logger.error(f"Error: {e}")
        return False
```

## 5. 프롬프트 체이닝 (A → B)

### 5.1 체이닝 개념도

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      프롬프트 체이닝 (A → B) 개념도                            │
└─────────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │                         analysis_main() 함수 내부                        │
  └─────────────────────────────────────────────────────────────────────────┘

  [Prompt A]                                    [Prompt B]
  ─────────                                    ─────────
       │                                            ▲
       │ 1) LLM 호출                                │
       ▼                                            │
  ┌──────────────┐                                  │
  │ result_A =   │                                  │
  │ ollama.      │                                  │
  │ generate(A)  │                                  │
  └──────┬───────┘                                  │
         │                                          │
         │ 2) JSON 파싱                             │
         ▼                                          │
  ┌──────────────┐                                  │
  │ result_A_json│                                  │
  │ = json_load()│                                  │
  └──────┬───────┘                                  │
         │                                          │
         │ 3) 결과값 추출                            │
         ▼                                          │
  ┌──────────────────────────────┐                  │
  │ value_from_A =               │                  │
  │   result_A_json.get("key")   │──────────────────┘
  └──────────────────────────────┘     4) B 프롬프트에
                                          플레이스홀더로 삽입
         │
         ▼
  ┌──────────────────────────────────────────┐
  │ new_prompt_B = self.prompt_B             │
  │   .replace("[A_result]", value_from_A)   │
  │   .replace("[contents]", content)        │
  └──────────────────────────────────────────┘
         │
         │ 5) B 프롬프트 LLM 호출
         ▼
  ┌──────────────┐
  │ result_B =   │
  │ ollama.      │
  │ generate(B)  │
  └──────────────┘
```

### 5.2 실제 코드 구조 (현재 시스템의 체이닝 예시)

현재 시스템에서 체이닝이 사용되는 곳:

```
question_verify (Step 1)
        │
        │ ai_keywords, related 추출
        ▼
question_summary (Step 2)
        │
        │ short_summary, long_summary 추출
        ▼
question_sentiment_scenario_2 (Step 3)
        │
        │ scenario2_* 결과 추출
        ▼
question_total_keywords_and_classification (Step 4)
```

### 5.3 체이닝 구현 패턴

#### 패턴 1: 단순 값 전달

```python
# A 프롬프트 결과에서 값 추출
result_A_json = {"positiveRatio": 70, "negativeRatio": 20}
positive_ratio = result_A_json.get("positiveRatio", "0")

# B 프롬프트에 삽입
new_prompt_B = self.prompt_B \
    .replace("[positiveRatio]", str(positive_ratio))
```

#### 패턴 2: 리스트 전달

```python
# A 프롬프트 결과에서 키워드 리스트 추출
result_A_json = {"keywords": ["혁신", "성장", "투자"]}
keywords = result_A_json.get("keywords", [])

# B 프롬프트에 삽입 (문자열로 변환)
keywords_str = ", ".join(keywords)
new_prompt_B = self.prompt_B \
    .replace("[extracted_keywords]", keywords_str)
```

#### 패턴 3: JSON 전체 전달

```python

import json
result_A_json = {"key1": "value1", "key2": "value2"}
result_A_str = json.dumps(result_A_json, ensure_ascii=False)

    .replace("[previous_analysis]", result_A_str)

def analysis_main(self, ...):
    # ... 기존 코드 ...
    
    # ========== 시나리오 2 실행 ==========
    result_sentiment_scenario_2 = self.chat_ollama._client.generate(...)
    is_success, result_sentiment_scenario_2_json = self.json_load(...)
    self.assemble_sentiment_scenario_2_to_model(...)
    
    # ========== 체이닝: 시나리오 2 결과를 새 프롬프트에 전달 ==========
    # 1) 시나리오 2 결과에서 값 추출
    s2_positive_ratio = result_sentiment_scenario_2_json.get("scenario2_positiveRatio", "0")
    s2_negative_ratio = result_sentiment_scenario_2_json.get("scenario2_negativeRatio", "0")
    s2_neutral_ratio = result_sentiment_scenario_2_json.get("scenario2_neutralRatio", "0")
    s2_positive_kw = result_sentiment_scenario_2_json.get("scenario2_positiveAllKeywords", [])
    s2_negative_kw = result_sentiment_scenario_2_json.get("scenario2_negativeAllKeywords", [])
    
    # 2) 새 프롬프트 플레이스홀더 치환
    new_question_detail = self.question_sentiment_detail \
        .replace("[organization]", str(orgName)) \
        .replace("[contents]", content) \
        .replace("[positiveRatio]", str(s2_positive_ratio)) \
        .replace("[negativeRatio]", str(s2_negative_ratio)) \
        .replace("[neutralRatio]", str(s2_neutral_ratio)) \
        .replace("[positiveKeywords]", ", ".join(s2_positive_kw)) \
        .replace("[negativeKeywords]", ", ".join(s2_negative_kw))
    
    # 3) LLM 호출
    result_detail = self.chat_ollama._client.generate(
        model=CONF.OLLAMA_MODEL,
        prompt=new_question_detail,
        format=(None if "gpt-oss" in CONF.OLLAMA_MODEL else "json")
    )
    
    # 4) 파싱 및 저장
    is_success, result_detail_json = self.json_load(result_detail, mycontents_logger)
    self.assemble_sentiment_detail_to_model(queueContent, orgName, result_detail, 
                                             result_detail_json, contentsMetaResult, mycontents_logger)
```

## 부록: 자주 발생하는 오류

| 오류 | 원인 | 해결 |
|:-----|:-----|:-----|
| `KeyError: 'newKey'` | VO에 필드 추가 안함 | `contentsVO.py`에 필드 추가 |
| JSON 파싱 실패 | 프롬프트 JSON 형식 오류 | `{{`, `}}` 이스케이프 확인 |
| 플레이스홀더 치환 안됨 | replace 문자열 불일치 | 대소문자, 공백 확인 |

| 체이닝 시 None 오류 | A 프롬프트 실패 | try-except로 예외 처리 |
